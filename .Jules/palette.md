## 2026-04-21 - Gradio Contextual Instructions vs Wall-of-Text
**Learning:** Consolidating app instructions into wall-of-text Markdown blocks creates visual clutter and reduces discoverability, as users often scan past long paragraphs before engaging with the interface. In Gradio, using contextual `info` fields attached directly to relevant input components places instructions exactly where users need them, making the UI cleaner and more accessible. However, since components like `gr.File` do not support the `info` property, adding a localized `gr.Markdown` description directly above them is the best fallback.
**Action:** Always prefer the `info` property on Gradio input components (like Slider or Textbox) to present help text instead of placing it in large, detached instructional blocks at the top of the interface. When a component lacks `info` support, position a brief `gr.Markdown` directly adjacent to it.
